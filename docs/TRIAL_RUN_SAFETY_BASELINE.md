# 试运行安全基线（2026-03-06）

适用范围：本地/测试环境联调，目标是“先稳定验证，再逐步放量”。

## 1. 环境基线

确保以下项已配置：

- `.env` 已配置 `JWT_SECRET`
- `.env` 已配置 `BACKUP_DIR`、`BACKUP_RETENTION_DAYS`、`BACKUP_SCHEDULE`
- PostgreSQL 与 Redis 已启动
- 至少一个通知通道可用（Telegram 或飞书）

推荐启动顺序：

```bash
pnpm dcup
pnpm dev
pnpm dev:frontend
```

## 2. 交易安全基线

试运行阶段建议采用以下策略：

- 单用户、单策略启动（先只开 1 个定投策略）
- 每笔小额（建议 50~100 元）
- 暂不开网格/再平衡，待定投 + 确认链路稳定后再开
- 保持止盈止损策略 `enabled=false`，先观察 1~2 个交易日

## 3. 验证清单（首日）

按顺序验证：

1. `GET /api/health` 返回可用
2. 登录后可正常获取 token，鉴权接口可访问
3. 手动触发一次净值同步（或等待调度任务）
4. 创建 1 条小额定投策略并确认调度执行日志
5. 检查通知是否送达
6. 次日检查 T+1 确认与持仓刷新是否正常
7. 检查备份目录是否生成备份文件

## 4. 观察指标（前 3 天）

- 调度任务成功率（尤其 14:30 定投、21:00 确认）
- 交易状态流转（`PENDING -> CONFIRMED/CANCELLED`）
- 通知送达率
- 队列是否积压（Redis/Bull）
- 备份是否按计划执行

## 5. 放量条件（满足后再扩大）

满足以下条件再增加策略与金额：

- 连续 3 个交易日无关键错误
- 定投执行与 T+1 确认链路稳定
- 告警与通知可及时送达
- 每日备份正常生成且可恢复验证

放量建议：

1. 从 1 个策略提升到 2~3 个策略
2. 单笔金额每周递增，不超过 20%~30%
3. 每次放量后至少观察 2 个交易日

## 6. 安全提醒

- 当前凭证均在本地 `.env` 明文，请确保 `.env` 不进入版本控制
- 若凭证曾在外部渠道暴露，建议立即轮换（交易账号、机器人 token、JWT secret）
